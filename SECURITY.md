# Security Policy

## Supported Versions

| Version | Supported |
|:--------|:---------:|
| 1.0.x   | Yes       |

Older versions are not supported. Please upgrade to the latest release.

---

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report privately via one of:

- **GitHub Security Advisories** — [New advisory](https://github.com/sudohakan/claude-remote-bot/security/advisories/new) (preferred)
- **Email** — hakan@finekra.com with subject `[SECURITY] claude-remote-bot`

Include in your report:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

You will receive an acknowledgment within **48 hours** and a resolution timeline within **7 days**. Critical issues targeting authentication, credential leakage, or remote code execution will be prioritized.

## Security Model

### Authentication

- **Invite-token gating** — Every user must redeem a single-use invite token before the bot processes any message. Tokens are generated by the admin via `/invite` and stored as hashed values.
- **Role hierarchy** — `admin` (set by `ADMIN_TELEGRAM_ID`), `user`, and `viewer`. Roles are enforced at the handler level before any business logic runs.
- **Telegram identity** — User identity is bound to the Telegram `user_id` and `chat_id`. There is no separate password or session cookie.

### Rate Limiting

Three independent per-user sliding-window rate limits prevent abuse:

| Limit | Default | Variable |
|:------|:-------:|:---------|
| Claude requests | 20 / min | `RATE_LIMIT_CLAUDE_PER_MIN` |
| Bot commands | 5 / min | `RATE_LIMIT_COMMANDS_PER_MIN` |
| Invite generation | 3 / hour | `RATE_LIMIT_INVITES_PER_HOUR` |

### Credential Sanitization

All Claude output passes through `CredentialSanitizer` before being sent to any Telegram chat. Patterns detected and redacted:

- AWS access/secret keys
- GitHub personal access tokens
- OpenAI API keys (`sk-...`)
- Anthropic API keys (`sk-ant-...`)
- PEM private key blocks
- Generic `Bearer` tokens in common formats

### Path Safety

The git integration enforces an allowlist-based path validator. All file reads and git operations are restricted to approved base directories. Directory traversal (`../`) and symlink escapes are rejected before reaching the filesystem.

### Cost Controls

`CostTracker` enforces two caps to prevent runaway API spend:

- **Per-request cap** — `CLAUDE_MAX_COST_PER_REQUEST` (default $1.00). Requests exceeding this are aborted mid-stream.
- **Per-user monthly cap** — `CLAUDE_MAX_COST_PER_USER` (default $5.00). Users over the cap are blocked until the next calendar month.

### Audit Log

Every command invocation and Claude interaction is written to the SQLite audit table with: timestamp, Telegram user ID, action type, and outcome. This log is append-only from the application's perspective.

### Secrets Management

- All secrets are loaded from environment variables or `.env` — never hardcoded.
- `.env` is in `.gitignore` by default. Do not commit it.
- The watchdog script (`wsl-watchdog.ps1`) reads credentials from Windows environment variables set by `install.sh`, not from files on disk.

### Transport Security

- All communication with the Telegram Bot API uses HTTPS (enforced by the library).
- ngrok tunnels use TLS by default. The ngrok authtoken is treated as a secret.

## Dependency Security

Production dependencies are pinned in `pyproject.toml`. Run the following to audit for known vulnerabilities:

```bash
pip install pip-audit
pip-audit
```

Update dependencies regularly and review the [Anthropic SDK changelog](https://github.com/anthropics/anthropic-sdk-python/releases) for security advisories.
